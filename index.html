What is this ?

This page is a listing of a lot of Hotspot JVM Option that I found in the source of the JVM.

It is based on the article "Hotspot JVM Options - The complete reference" that I published on Pingtimeout.fr. After I published it, I was convinced that a better form could be use to present all this information, so I made an application out of it.

This application is released under the GPL v3, and its source code is available on Github.

What is the difference between -X and -XX ?

According to Oracle website,

  • Options that begin with -X are non-standard (not guaranteed to be supported on all VM implementations), and are subject to change without notice in subsequent releases of the JDK.
  • Options that are specified with -XX are not stable and are subject to change without notice.

So even if highly improbable, according to this notice, options like -Xmx512m might be deprecated or changed *without notice* and options like -XX:MaxPermSize might not be stable.

Options category

Each option comes with a specific category, which meaning is the following :

  • The options categorized as Product can be used with the command line argument -XX:
  • The options categorized as Diagnostic can be used with the command line argument -XX: if the argument -XX:+UnlockDiagnosticOption is placed before them
  • The options categorized as Develop are settable in a development Hotspot JVM (debug or fastdebug build) using the command line argument -XX:. In the standard (generally available JVM), those options are constant

Options architecture

Each option comes with one or two labels indicating the architecture on which it is available :

  • 64-bits means that the option is available on 64-bits platforms
  • 32-bits means that the option is available on 32-bits platforms
